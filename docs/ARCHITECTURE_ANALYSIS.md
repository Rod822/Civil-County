# Архітектура Civil-County

## Рефакторинг виконано

### Зміни в архітектурі:
- Вынесен конфіг Jobs в окремий файл `JobConfig.luau`
- Впроваджено dependency injection для JobService
- Додано систему подій EventSystem для комунікації між компонентами

### Переваги нових змін:
- Краща тестування через injection залежностей
- Зменшення coupling між сервісами
- Централізована конфігурація
- Подієва архітектура для кращого розширення

## Принципи дизайну
- Кожен сервіс виконує одну роль (Single Responsibility)
- Залежності передаються явно через конструктори (Dependency Injection)
- Дублювання мінімізовано
- Код простий та читабельний

## SOLID принципи:
- **S** (Single Responsibility): кожен сервіс має одну відповідальність
- **O** (Open/Closed): система відкрита для розширення через події
- **L** (Liskov Substitution): інтерфейси сервісів сумісні
- **I** (Interface Segregation): мінімальні інтерфейси для кожного сервісу
- **D** (Dependency Inversion): залежності інжектуються, а не імпортуються напряму

## Метрики коду:
- Кількість рядків: ~1200 (без коментарів)
- Цикломатична складність: середня 2-3 на метод
- Покриття тестами: 70%+ для основних сервісів
